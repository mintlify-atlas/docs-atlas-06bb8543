The Groq integration provides access to blazing-fast language model inference using Groq's proprietary Language Processing Unit (LPU) technology.

## Authentication

You need a Groq API key to use this integration.

### Getting Your API Key

1. Go to [GroqCloud](https://console.groq.com/)
2. Sign up or log in to your account
3. Navigate to "API Keys"
4. Click "Create API Key"
5. Give your key a descriptive name
6. Copy the API key immediately
7. Store it securely

### Adding to Flowbaker

1. Navigate to your workspace settings
2. Go to "Credentials" → "Add Credential"
3. Select "Groq"
4. Paste your API key
5. Save the credential

## What Makes Groq Different?

### LPU Technology

Groq uses Language Processing Units (LPUs) instead of traditional GPUs:

- **Extremely fast inference**: 10-100x faster than GPU-based solutions
- **Low latency**: Sub-second response times
- **Deterministic performance**: Consistent, predictable speeds
- **Energy efficient**: Lower power consumption

### Speed Benefits

- **Tokens per second**: Up to 500+ tokens/second
- **Time to first token**: < 100ms
- **Batch processing**: Massive throughput
- **Real-time applications**: Perfect for interactive use

## Available Actions

### AI Agent Chat

Use Groq models within AI agents for ultra-fast autonomous task completion.

**Parameters:**
- `model` (required): The Groq model to use
- `temperature` (optional): Creativity level (0-2, default: varies)
- `max_tokens` (optional): Maximum tokens to generate
- `top_p` (optional): Nucleus sampling parameter (0-1)

**Example:**
```json
{
  "model": "groq/compound",
  "temperature": 0.7,
  "max_tokens": 2048,
  "top_p": 0.9
}
```

**Note:** This action is designed for use within AI Agent nodes and supports tool calling.

## Error Handling

### Common Errors

**"Invalid API key"**
- Verify your API key is correct
- Check it hasn't been revoked

**"Rate limit exceeded"**
- Wait for limit to reset
- Consider upgrading your plan

**"Model not available"**
- Check model name spelling
- Some models may have limited availability

**"Service temporarily unavailable"**
- Groq is experiencing high load
- Retry with exponential backoff

## Limitations

### Current Limitations

1. **Model selection**: Fewer models than GPT/Claude
2. **Context windows**: Smaller than some competitors (except Kimi)
3. **Multimodal**: Currently text-only
4. **Fine-tuning**: Limited custom model options

### When to Use Alternatives

- **Very long context**: Use Gemini (2M tokens)
- **Image generation**: Use DALL-E or Midjourney
- **Image analysis**: Use GPT-4V or Gemini
- **Maximum capability**: Use GPT-4 or Claude Opus
