The Flowbaker Storage integration provides built-in file storage and management within your workspace, allowing you to persist files generated during workflow execution.

## Authentication

No authentication required - this is an internal service that automatically uses your workspace credentials.

## Available Actions

### List Files
List files in your workspace with optional filtering.

**Parameters:**
- `limit` (optional): Maximum files to return (1-1000, default: 1000)
- `folder_id` (optional): Folder to list from (empty for root)
- `from_file_id` (optional): Continue listing from this file ID (pagination)

### Get File
Retrieve a file and its content.

**Parameters:**
- `file_id` (required): Unique identifier of the file

### Delete File
Permanently remove a file from workspace storage.

**Parameters:**
- `file_id` (required): Unique identifier of the file to delete

### Persist File
Remove expiration date from a file to prevent automatic deletion.

**Parameters:**
- `file` (required): File to persist (remove expiration)

## Configuration Example

```json
{
  "limit": 100,
  "folder_id": "folder_abc123"
}
```

## File Lifecycle

Flowbaker Storage manages file lifecycle automatically:

1. **Temporary Files**: Files generated during workflow execution are temporary by default
2. **Expiration**: Temporary files expire after a set period
3. **Persistence**: Use "Persist File" action to prevent expiration
4. **Manual Deletion**: Delete files explicitly when no longer needed

## File Organization

- Files can be organized into folders
- Use folder IDs to navigate the hierarchy
- Root folder is accessed by leaving `folder_id` empty
- Folders are created automatically when needed

## Pagination

When listing files:
1. Set a `limit` to control results per request
2. Check the response for more files
3. Use `from_file_id` from the last file to continue
4. Repeat until all files are retrieved

## Best Practices

1. **Persist Important Files**: Use "Persist File" for files you want to keep
2. **Clean Up**: Delete files when no longer needed to save storage
3. **Organize**: Use folders to organize files logically
4. **Pagination**: Use pagination for large file lists
5. **Error Handling**: Handle cases where files may not exist
6. **File Naming**: Use descriptive file names for easy identification

## Storage Limits

Flowbaker Storage has workspace-level limits:
- Total storage capacity varies by plan
- Individual file size limits apply
- Check your workspace settings for current limits

## Common Use Cases

- **Report Storage**: Store generated reports and documents
- **Data Export**: Export workflow data to files
- **Temporary Processing**: Store intermediate processing results
- **File Transfer**: Transfer files between workflow steps
- **Backup**: Keep copies of important generated files
- **Log Files**: Store execution logs and audit trails

## Integration with Workflows

Flowbaker Storage integrates seamlessly:
- Files generated by actions can be automatically stored
- Reference stored files in subsequent workflow steps
- Pass files between different integrations
- Persist results from long-running workflows

## File Metadata

Stored files include metadata:
- File ID (unique identifier)
- File name
- File size
- Content type (MIME type)
- Creation timestamp
- Expiration date (if applicable)
- Folder location

## Example Workflow

1. Generate a report using an integration
2. Store the report using Flowbaker Storage
3. Persist the file to prevent expiration
4. Reference the file in email notifications
5. Clean up old reports periodically
