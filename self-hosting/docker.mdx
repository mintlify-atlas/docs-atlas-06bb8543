Docker is the recommended way to deploy the Flowbaker executor. This guide covers building and running the executor using Docker.

## Quick Start

<Steps>
  <Step title="Pull or build the image">
    Build the Docker image from source:
    
    ```bash
    docker build -t flowbaker-executor .
    ```
  </Step>

  <Step title="Run the container">
    ```bash
    docker run -d \
      --name flowbaker-executor \
      -p 8081:8081 \
      -v flowbaker-config:/home/flowbaker/.flowbaker \
      flowbaker-executor
    ```
  </Step>

  <Step title="Complete initial setup">
    On first run, attach to the container to complete the interactive setup:
    
    ```bash
    docker logs -f flowbaker-executor
    ```
    
    The executor will guide you through registration and configuration.
  </Step>
</Steps>

## Dockerfile Overview

The Flowbaker executor uses a multi-stage Dockerfile for optimized builds:

### Build Stage

```dockerfile
# Build stage
FROM golang:1.25.0-alpine AS builder

# Build arguments
ARG VERSION=dev
ARG GIT_COMMIT=""
ARG BUILD_DATE=""
ARG BUILD_USER=""

# Install git and ca-certificates for dependency fetching
RUN apk add --no-cache git ca-certificates

# Set working directory
WORKDIR /app

# Copy go mod files
COPY go.mod go.sum ./

# Download dependencies
RUN go mod download

# Copy source code
COPY . .

# Build the application with version information
RUN CGO_ENABLED=0 GOOS=linux go build -a -installsuffix cgo \
    -ldflags "-X github.com/flowbaker/flowbaker/internal/version.Version=${VERSION} \
              -X github.com/flowbaker/flowbaker/internal/version.GitCommit=${GIT_COMMIT} \
              -X github.com/flowbaker/flowbaker/internal/version.BuildDate=${BUILD_DATE} \
              -X github.com/flowbaker/flowbaker/internal/version.BuildUser=${BUILD_USER}" \
    -o flowbaker ./cmd
```

**Key Features:**
- Uses Go 1.25.0 for compilation
- Static binary with `CGO_ENABLED=0` for portability
- Version information embedded via build flags
- Dependency caching for faster rebuilds

### Runtime Stage

```dockerfile
# Final stage
FROM alpine:latest

# Install ca-certificates for HTTPS requests
RUN apk --no-cache add ca-certificates

# Create app directory and non-root user
RUN mkdir /app && \
    addgroup -g 1001 flowbaker && \
    adduser -D -u 1001 -G flowbaker flowbaker

WORKDIR /app

# Copy the binary from builder stage
COPY --from=builder /app/flowbaker .

# Change ownership to non-root user
RUN chown -R flowbaker:flowbaker /app

# Switch to non-root user
USER flowbaker

# Expose port
EXPOSE 8081

# Health check
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
    CMD wget --no-verbose --tries=1 --spider http://localhost:8081/health || exit 1

# Default command
CMD ["./flowbaker", "start"]
```

**Security Features:**
- Minimal Alpine Linux base image
- Non-root user (`flowbaker:1001`)
- Read-only binary execution
- Built-in health check

## Building the Image

### Basic Build

```bash
docker build -t flowbaker-executor .
```

### Build with Version Information

```bash
docker build \
  --build-arg VERSION=1.0.0 \
  --build-arg GIT_COMMIT=$(git rev-parse HEAD) \
  --build-arg BUILD_DATE=$(date -u +"%Y-%m-%dT%H:%M:%SZ") \
  --build-arg BUILD_USER=$(whoami) \
  -t flowbaker-executor:1.0.0 \
  .
```

### Multi-platform Build

Build for multiple architectures using buildx:

```bash
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  -t flowbaker-executor:latest \
  .
```

## Running the Container

### Basic Run

```bash
docker run -d \
  --name flowbaker-executor \
  -p 8081:8081 \
  flowbaker-executor
```

### Run with Persistent Configuration

```bash
docker run -d \
  --name flowbaker-executor \
  -p 8081:8081 \
  -v flowbaker-config:/home/flowbaker/.flowbaker \
  flowbaker-executor
```

<Note>
Using a named volume ensures configuration persists across container restarts.
</Note>

### Run with Environment Variables

```bash
docker run -d \
  --name flowbaker-executor \
  -p 8081:8081 \
  -e FLOWBAKER_EXECUTOR_NAME="production-executor" \
  -e FLOWBAKER_EXECUTOR_ADDRESS="https://executor.example.com" \
  -e FLOWBAKER_API_URL="https://api.flowbaker.io" \
  -e STATIC_API_SIGNATURE_PUBLIC_KEY="your-public-key" \
  flowbaker-executor
```

### Run with Custom Port

```bash
docker run -d \
  --name flowbaker-executor \
  -p 9000:8081 \
  flowbaker-executor
```

<Warning>
If you change the host port, update `FLOWBAKER_EXECUTOR_ADDRESS` to reflect the correct external URL.
</Warning>

## Docker Compose

For production deployments, use Docker Compose:

### Basic Compose File

```yaml
version: '3.8'

services:
  executor:
    image: flowbaker-executor:latest
    build:
      context: .
      dockerfile: Dockerfile
      args:
        VERSION: "1.0.0"
        GIT_COMMIT: "${GIT_COMMIT}"
        BUILD_DATE: "${BUILD_DATE}"
    container_name: flowbaker-executor
    ports:
      - "8081:8081"
    volumes:
      - flowbaker-config:/home/flowbaker/.flowbaker
    environment:
      - FLOWBAKER_EXECUTOR_NAME=production-executor
      - FLOWBAKER_API_URL=https://api.flowbaker.io
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "wget", "--no-verbose", "--tries=1", "--spider", "http://localhost:8081/health"]
      interval: 30s
      timeout: 3s
      start_period: 5s
      retries: 3

volumes:
  flowbaker-config:
    driver: local
```

### Production Compose File

```yaml
version: '3.8'

services:
  executor:
    image: flowbaker-executor:${VERSION:-latest}
    build:
      context: .
      dockerfile: Dockerfile
      args:
        VERSION: "${VERSION:-dev}"
        GIT_COMMIT: "${GIT_COMMIT}"
        BUILD_DATE: "${BUILD_DATE}"
    container_name: flowbaker-executor
    ports:
      - "8081:8081"
    volumes:
      - flowbaker-config:/home/flowbaker/.flowbaker
      - /var/run/docker.sock:/var/run/docker.sock:ro  # If workflows need Docker access
    environment:
      # Core Configuration
      - FLOWBAKER_EXECUTOR_NAME=${EXECUTOR_NAME:-flowbaker-executor}
      - FLOWBAKER_EXECUTOR_ADDRESS=${EXECUTOR_ADDRESS}
      - FLOWBAKER_API_URL=${API_URL:-https://api.flowbaker.io}
      
      # Security
      - STATIC_API_SIGNATURE_PUBLIC_KEY=${API_SIGNATURE_PUBLIC_KEY}
      - FLOWBAKER_ENABLE_WORKSPACE_REGISTRATION=${ENABLE_WORKSPACE_REGISTRATION:-true}
      
      # Pre-configured Keys (Optional)
      - FLOWBAKER_EXECUTOR_ID=${EXECUTOR_ID}
      - FLOWBAKER_X25519_PRIVATE_KEY=${X25519_PRIVATE_KEY}
      - FLOWBAKER_X25519_PUBLIC_KEY=${X25519_PUBLIC_KEY}
      - FLOWBAKER_ED25519_PRIVATE_KEY=${ED25519_PRIVATE_KEY}
      - FLOWBAKER_ED25519_PUBLIC_KEY=${ED25519_PUBLIC_KEY}
    env_file:
      - .env
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "wget", "--no-verbose", "--tries=1", "--spider", "http://localhost:8081/health"]
      interval: 30s
      timeout: 3s
      start_period: 5s
      retries: 3
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"
    security_opt:
      - no-new-privileges:true
    read_only: true
    tmpfs:
      - /tmp

volumes:
  flowbaker-config:
    driver: local
```

### Environment File (.env)

Create a `.env` file for sensitive configuration:

```bash
# .env
VERSION=1.0.0
EXECUTOR_NAME=production-executor
EXECUTOR_ADDRESS=https://executor.example.com
API_URL=https://api.flowbaker.io

# Security Keys
API_SIGNATURE_PUBLIC_KEY=your-public-key-here
EXECUTOR_ID=your-executor-id
X25519_PRIVATE_KEY=your-x25519-private-key
X25519_PUBLIC_KEY=your-x25519-public-key
ED25519_PRIVATE_KEY=your-ed25519-private-key
ED25519_PUBLIC_KEY=your-ed25519-public-key

# Build Info
GIT_COMMIT=$(git rev-parse HEAD)
BUILD_DATE=$(date -u +"%Y-%m-%dT%H:%M:%SZ")
```

<Warning>
Never commit the `.env` file to version control. Add it to `.gitignore`.
</Warning>

## Container Management

### Start the Service

```bash
docker-compose up -d
```

### View Logs

```bash
docker-compose logs -f executor
```

### Check Health Status

```bash
docker-compose ps
```

### Restart the Service

```bash
docker-compose restart executor
```

### Stop the Service

```bash
docker-compose down
```

### Update and Rebuild

```bash
docker-compose down
docker-compose build --no-cache
docker-compose up -d
```

## Health Checks

The Dockerfile includes a built-in health check:

```dockerfile
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
    CMD wget --no-verbose --tries=1 --spider http://localhost:8081/health || exit 1
```

**Health Check Parameters:**
- **Interval**: Check every 30 seconds
- **Timeout**: 3 seconds per check
- **Start Period**: 5 seconds grace period on startup
- **Retries**: Mark unhealthy after 3 consecutive failures

### Manual Health Check

```bash
docker exec flowbaker-executor wget -qO- http://localhost:8081/health
```

## Volume Management

### Inspect Configuration Volume

```bash
docker volume inspect flowbaker-config
```

### Backup Configuration

```bash
docker run --rm \
  -v flowbaker-config:/source:ro \
  -v $(pwd):/backup \
  alpine tar czf /backup/flowbaker-config-backup.tar.gz -C /source .
```

### Restore Configuration

```bash
docker run --rm \
  -v flowbaker-config:/target \
  -v $(pwd):/backup:ro \
  alpine tar xzf /backup/flowbaker-config-backup.tar.gz -C /target
```

## Security Considerations

### Run as Non-Root User

The Dockerfile automatically creates and uses a non-root user:

```dockerfile
USER flowbaker
```

### Read-Only Filesystem

For enhanced security, run with a read-only filesystem:

```bash
docker run -d \
  --name flowbaker-executor \
  --read-only \
  --tmpfs /tmp \
  -v flowbaker-config:/home/flowbaker/.flowbaker \
  flowbaker-executor
```

### Network Security

```bash
docker network create flowbaker-network

docker run -d \
  --name flowbaker-executor \
  --network flowbaker-network \
  -p 127.0.0.1:8081:8081 \
  flowbaker-executor
```

### Security Options

```bash
docker run -d \
  --name flowbaker-executor \
  --security-opt no-new-privileges \
  --cap-drop ALL \
  flowbaker-executor
```

## Troubleshooting

### Container Won't Start

Check logs:
```bash
docker logs flowbaker-executor
```

### Port Already in Use

Change the host port:
```bash
docker run -p 9000:8081 flowbaker-executor
```

### Configuration Issues

Reset configuration volume:
```bash
docker-compose down -v
docker-compose up -d
```

### Access Container Shell

```bash
docker exec -it flowbaker-executor sh
```
