## Overview

The `Workflow` type represents a complete workflow definition in Flowbaker. Workflows are composed of nodes that define the automation logic.

## Workflow Structure

### Go Definition

```go
type Workflow struct {
    ID               string
    Name             string
    Description      string
    Slug             string
    WorkspaceID      string
    AuthorUserID     string
    Nodes            []WorkflowNode
    Settings         WorkflowSettings
    LastUpdatedAt    time.Time
    ActivationStatus WorkflowActivationStatus
    DeletedAt        *time.Time
}
```

## Helper Methods

### IsActive

Checks if the workflow is currently active.

```go
func (w Workflow) IsActive() bool
```

**Example:**
```go
if workflow.IsActive() {
    // Execute the workflow
}
```

### GetNodeByID

Retrieves a specific node by its ID.

```go
func (w Workflow) GetNodeByID(nodeID string) (WorkflowNode, bool)
```

**Example:**
```go
node, found := workflow.GetNodeByID("node-123")
if found {
    // Process the node
}
```

### GetTriggerNodes

Returns all trigger nodes in the workflow.

```go
func (w Workflow) GetTriggerNodes() []WorkflowNode
```

**Example:**
```go
triggers := workflow.GetTriggerNodes()
for _, trigger := range triggers {
    // Process each trigger
}
```

### GetActionNodes

Returns all action nodes in the workflow.

```go
func (w Workflow) GetActionNodes() []WorkflowNode
```

**Example:**
```go
actions := workflow.GetActionNodes()
for _, action := range actions {
    // Process each action
}
```

### GetSubNodes

Returns all nodes that are children of a specific parent node.

```go
func (w Workflow) GetSubNodes(nodeID string) []WorkflowNode
```

**Example:**
```go
subNodes := workflow.GetSubNodes("parent-node-123")
```

## Usage in API Requests

Workflows are passed in API requests for execution:

```bash
curl -X POST \
  http://localhost:8080/workspaces/ws-xyz789/executions \
  -H 'Content-Type: application/json' \
  -d '{
    "execution_id": "exec-123",
    "workflow": {
      "id": "wf-abc123",
      "name": "Customer Onboarding",
      "workspace_id": "ws-xyz789",
      "nodes": [...],
      "settings": {
        "node_execution_limit": 100
      },
      "activation_status": "active"
    },
    "event_name": "customer.created",
    "payload_json": "..."
  }'
```
