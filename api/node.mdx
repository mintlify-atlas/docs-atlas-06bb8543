## Overview

Nodes are the building blocks of workflows. Each node represents either a trigger (event source) or an action (operation) in the workflow automation.

## Node Structure

### Go Definition

```go
type WorkflowNode struct {
    ID                           string
    WorkflowID                   string
    Name                         string
    Type                         NodeType
    IntegrationType              IntegrationType
    SubscribedEvents             []string
    Positions                    NodePositions
    IntegrationSettings          map[string]any
    Settings                     NodeSettings
    ExpressionSelectedProperties []string
    ProvidedByAgent              []string
    Inputs                       []NodeInput
    UsageContext                 string
    ParentID                     string
    TriggerNodeOpts              TriggerNodeOpts
    ActionNodeOpts               ActionNodeOpts
}
```

<ParamField path="trigger" type="NodeType">

</ParamField>

<ParamField path="action" type="NodeType">

## Node Settings

```go
type NodeSettings struct {
    ReturnErrorAsItem       bool
    OverwriteExecutionLimit bool
    ExecutionLimit          int
}
```

<ParamField path="ReturnErrorAsItem" type="bool">
  If true, errors are returned as regular items instead of stopping execution
</ParamField>

<ParamField path="OverwriteExecutionLimit" type="bool">
  If true, use this node's execution limit instead of the workflow's default
</ParamField>

<ParamField path="ExecutionLimit" type="int">
  Maximum number of times this node can execute in a single workflow run
</ParamField>

## Node Execution Flow

1. **Trigger nodes** initiate workflow execution when their events occur
2. Data flows through **subscribed events** to connected nodes
3. **Action nodes** execute when they receive data from their inputs
4. **Expression-selected properties** allow dynamic data interpolation
5. **Execution limits** prevent infinite loops
