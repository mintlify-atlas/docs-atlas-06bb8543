# Integrations

Integrations connect Flowbaker to external services and platforms. Each integration provides triggers to start workflows and actions to perform operations.

## Integration Structure

An integration is defined by the `Integration` struct:

```go
type Integration struct {
    ID                   IntegrationType
    Name                 string
    Description          string
    CredentialProperties []NodeProperty
    Actions              []IntegrationAction
    Triggers             []IntegrationTrigger
    EmbeddingModels      []IntegrationEmbeddingModel
    IsGroup              bool
    CanTestConnection    bool
    IsCredentialOptional bool
}
```

### Key Properties

- **ID**: Unique identifier for the integration type
- **Name**: Display name of the integration
- **Description**: What the integration does
- **CredentialProperties**: Required authentication properties
- **Actions**: Operations this integration can perform
- **Triggers**: Events this integration can listen for
- **CanTestConnection**: Whether credentials can be validated
- **IsCredentialOptional**: Whether authentication is required

## Integration Actions

Actions are operations that an integration can perform:

```go
type IntegrationAction struct {
    ID                            string
    ActionType                    IntegrationActionType
    Name                          string
    Description                   string
    Properties                    []NodeProperty
    HandlesByContext              map[ActionUsageContext]ContextHandles
    SupportedContexts             []ActionUsageContext
    CombinedContexts              []ActionUsageContext
    IsNonAvailableForDefaultOAuth bool
    Decoration                    NodeDecoration
}
```

### Action Properties

- **ActionType**: Unique identifier for this action
- **Properties**: Configuration parameters the action requires
- **HandlesByContext**: Input/output handles for different usage contexts
- **SupportedContexts**: Where this action can be used (workflow, tool, etc.)
- **Decoration**: UI hints like buttons or disabled editor

### Finding Actions

You can retrieve a specific action from an integration:

```go
func (i Integration) GetActionByType(actionType IntegrationActionType) (IntegrationAction, bool)
```

## Integration Triggers

Triggers listen for events and start workflow execution:

```go
type IntegrationTrigger struct {
    ID                            string
    EventType                     IntegrationTriggerEventType
    Name                          string
    Description                   string
    Properties                    []NodeProperty
    HandlesByContext              map[ActionUsageContext]ContextHandles
    IsNonAvailableForDefaultOAuth bool
    Decoration                    NodeDecoration
}
```

### Trigger Properties

- **EventType**: Unique identifier for this trigger event
- **Properties**: Configuration parameters for the trigger
- **HandlesByContext**: Output handles that provide data to downstream nodes

### Finding Triggers

Retrieve a specific trigger from an integration:

```go
func (i Integration) GetTriggerByType(triggerType IntegrationTriggerEventType) (IntegrationTrigger, bool)
```

## Node Handles

Handles define the connection points for data flow between nodes:

```go
type NodeHandle struct {
    Index               int
    Type                NodeHandleType
    Position            NodeHandlePosition
    Text                string
    UsageContext        ActionUsageContext
    AllowedIntegrations []IntegrationType
}

type ContextHandles struct {
    Input  []NodeHandle
    Output []NodeHandle
}
```

### Handle Types

```go
const (
    NodeHandleTypeDefault     NodeHandleType = "default"
    NodeHandleTypeSuccess     NodeHandleType = "success"
    NodeHandleTypeDestructive NodeHandleType = "destructive"
)
```

- **default**: Standard data flow
- **success**: Executed on successful completion
- **destructive**: For error or failure paths

### Handle Positions

```go
const (
    NodeHandlePositionTop    NodeHandlePosition = "top"
    NodeHandlePositionBottom NodeHandlePosition = "bottom"
    NodeHandlePositionLeft   NodeHandlePosition = "left"
    NodeHandlePositionRight  NodeHandlePosition = "right"
)
```

Handle positions determine where connections appear on the node in visual editors.

## Integration Input & Output

When a node executes, it receives input and produces output:

### Integration Input

```go
type IntegrationInput struct {
    NodeID            string
    PayloadByInputID  map[string]Payload
    IntegrationParams IntegrationParams
    ActionType        IntegrationActionType
    Workflow          *Workflow
}

type IntegrationParams struct {
    Settings map[string]any
}
```

The input includes:
- **NodeID**: Which node is being executed
- **PayloadByInputID**: Data from upstream nodes
- **IntegrationParams**: Node configuration settings
- **ActionType**: Which action to perform
- **Workflow**: Reference to the parent workflow

### Integration Output

```go
type IntegrationOutput struct {
    ResultJSONByOutputID []Payload
}
```

The output contains an array of payloads, one for each output handle. These payloads flow to downstream nodes.

## Credentials

Many integrations require authentication credentials:

### Credential Properties

Integrations define which credential properties they need:

```go
CredentialProperties []NodeProperty
```

Common credential types include:
- API keys
- OAuth tokens
- Username/password combinations
- Service account credentials

### Testing Connections

Integrations with `CanTestConnection: true` allow you to validate credentials before using them in workflows.

<Warning>
Always test your integration credentials before activating workflows to ensure proper authentication.
</Warning>

These dependencies provide:

- Knowledge base access
- AI model management

## Usage Contexts

Integrations can be used in different contexts:

```go
type ActionUsageContext string

const (
    UsageContextWorkflow       ActionUsageContext = "workflow"        // Regular workflow automation
    UsageContextTool           ActionUsageContext = "tool"            // AI Agent tool
    UsageContextLLMProvider    ActionUsageContext = "llm_provider"    // LLM provider for AI agents
    UsageContextMemoryProvider ActionUsageContext = "memory_provider" // Memory provider for AI agents
)
```

This allows the same integration to serve multiple purposes:
- As workflow automation nodes
- As tools available to AI agents
- As AI providers (LLM, memory, embeddings)
